Test Case Description for view-properties

Purpose

This testcase is designed to verify that the view properties feature works as prescribed for various items.

Initial Conditions

  • You must have a bookmarks file.
  • To promote consistency, you may choose to use the default set of Netscape Seamonkey Bookmarks, which I use. I have also provided the default Mozilla bookmarks file.
  • You also have the option to use this very large bookmarks file.
  • You must also customize your sidebar to include a bookmarks panel.

Steps/Description

  1. Select Bookmarks|Manage Bookmarks.
  2. Select a bookmark. Select File|Bookmark Properties. Verify, click "OK".
  3. Select a folder. Select File|Bookmark Properties. Verify, click "OK".
  4. Select a separator. Select File|Bookmark Properties.

Expected Results

  1. For step(2) the Bookmark Properties dialog should appear. It should have fields for name, location(url), custom keyword, and description. There should also be a section for setting update and notification options.
  2. For step(3) the Bookmark Properties dialog should appear. It should have fields for name and description. No other fileds should appear in this dialog.
  3. For step(4) no dialog should appear.
