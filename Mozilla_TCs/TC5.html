Test Case Description for non-clickable items in Help

Purpose

This basic testcase is designed to verify that when clicking non-clickable areas in help, no behavior occurs

Initial Conditions

Netscape should be started up

Steps/Description

  1. Select the toplevel Help menu.
  2. Select Help Contents
  3. Click on the greyed out previous button
  4. Click on the greyed out back button
  5. Click on the Netscape logo in uppper right

Expected Results

2. You should be taken to the help contents page

3. Nothing should happen

4. Nothing should happen

5. Nothing should happen
