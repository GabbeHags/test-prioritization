Test Case Description for paste-into-folder

Purpose

This basic testcase is designed to verify that pasting into containers works properly.

Initial Conditions

  • You must have a bookmarks file.
  • To promote consistency, you may choose to use the default set of Netscape Seamonkey Bookmarks, which I use. I have also provided the default Mozilla bookmarks file.
  • You also have the option to use this very large bookmarks file.
  • You must also customize your sidebar to include a bookmarks panel.

Steps/Description

  1. Select Bookmarks|Manage Bookmarks.
  2. Select any item.
  3. Select Edit|Copy.
  4. Select any closed folder.
  5. Select Edit|Paste.
  6. Select any open folder (open a folder).
  7. Select Edit|Paste
  8. Repeat steps(2-7) using the same folder for step(2), step(4) and step(6) ie paste a folder into itself.

Expected Results

  1. All items should paste below a closed folder.
  2. All items except the identical folder should paste into an open folder.
  3. Pasting a folder into itself should not be allowed (*Communicator 4.x allows this).
  4. Newly pasted items should appear in the 4 standard locations.
